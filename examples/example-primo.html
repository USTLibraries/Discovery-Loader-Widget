Documentation

The js and css code is fully documented with comments.

What you will need:

  • A place to host your css and js files
  • A basic understanding of 3 HTML tags (DIV, A, SCRIPT), tag attributes (data-, href, src), and setting variables in JavaScript (Don't worry, it's all heavily commented)
  • jQuery and jQuery UI loaded on the page (works without autosuggest if no jQuery UI). If you need to add jQuery to your HEAD yourself: 
       <script src="https://ajax.googleapis.com/ajax/libs/jquery/2.1.3/jquery.min.js"></script>
   <script src="https://ajax.googleapis.com/ajax/libs/jqueryui/1.11.4/jquery-ui.min.js"></script>
  • It does fall back to plain JavaScript if no jQuery, but produces an unscoped basic search box.

IMPORTANT!: There is currently NOT an autosuggest feature though the code is there for future use. Therefore you do NOT need jQuery UI. In order to use autosuggest you will need to have an autosuggest service as Primo does not provide one like Summon does. I'm working on a solution.

Files:

  • primo.js
  • primo.css

Functionality:

  • Works on main website and LibGuides 2.0
  • Won't conflict with other widgets of the same kind if mulitple on one page (even if not unique element ID)
  • Degrades gracefully if no JavaScript (Display a default link to the discovery search page)
  • Degrades gracefully if no jQuery, falling back to a simple JavaScript generated html search form with no scoping [see example]
  • Degrades gracefully if no jQuery UI, and will not attempt autosuggest
  • If jQuery UI available, it will have an autosuggest
  • Enter key will issue search and on screen "Search" key on iOS and Android will initiate search
  • Compatible with Windows (IE, Chrome, Firefox), Mac (Safari, Chrome, Firefox), iOS (Safari, Chrome, Firefox), Android (Chrome, Firefox)
  • Responsive
  • Meets Accessibility standards (Has a label for screen readers and the submit button is marked)

Documentation that walks you through the code and logic may be found in the comments of the source of this page and the source of the scripts and css. So be sure to view the plain HTML source for a greater, technical, understanding.

The link within the <div> takes user to main search page: Search Library Resources and is used as placeholder text within the <div> tag. It will be replaced by the search form. Or, if JavaScript is not enabled, remains as a means of getting to the main search page.... on which JavaScript will be required. Note that this is the only piece of HTML that needs to be customized.

Installation

  1. Download and customize the primo.js file (see JavaScript Customization below)
  2. Download and customize the primo.css file (basically the color of the button at the very least unless you like purple)
  3. Place primo.js and primo.css files on your server
  4. Add DIV HTML, JavaScript src, and (optional) CSS link to your pages (the script will auto load CSS if not present).
  5. Be sure the links in your DIV tags point to your Primo search

Basic format that we copy and paste into your CMS is:

JavaScript in HEAD (or at end of BODY, or immediately following DIV):

<script type="text/javascript" src="../primo.js"></script>

CSS in HEAD (or include CSS within your main css, or not at all since the script will check for it an include it automatically):

<link href="primo.css" type="text/css">

HTML in BODY:

<div class="discovery-search-widget">
    <a href="https://your-primo.hosted.exlibrisgroup.com/primo-explore/search?vid=YOURVIEW">Search Library Resources</a>
</div>

What are the div attributes available?

I used the Primo Search Box Documentation, trial and error using the available facets, and previous experience with Matthew Reidsma's (GVSU) Custom Summon Searches to come up with these.

The nice thing is that the DIV attributes are interchangable between summon.js and primo.js which was the intent. The main difference is the way date scope is formatted (YYYY:YYYY for Primo, YYYY-MM-DD:YYYY-MM-DD for Summon) and the fact that Primo can only take 1 material type. Also, Primo allows added functionality such as Course Reserves and Journal Title search so additional options for data-scope are available.

div attributes:

All attributes are optional. you can do something as simple as:

<div class="discovery-search-widget">
    <a href="https://your-primo.hosted.exlibrisgroup.com/primo-explore/search?vid=YOURVIEW">Search Library Resources</a>
</div>

... or complex by using every attribute available... if you want.

id
[unique element id] - if it is not unique, the script will make it unique so there won't be conflicts
data-scope
author|title|journal|course|course_id|course_name|course_instr|course_dept - these will automatically add AuthorCombined() or TitleCombined() to the search query when submitted. Note that for journal searches all other data-scope- are ignored. Any data-scope value starting with "course" will be treated as a Course Reserve Search.
data-scope-content-type
BOOKS|VIDEO|AUDIO|MUSIC|MEDIA|[other facet value] - BOOKS, VIDEO, AUDIO, MUSIC, and MEDIA (all upper) are predefined but customizable (see JavaScript Customization section below): BOOKS = "books"; VIDEO = "video"; AUDIO = "audio"; MUSIC = "scores"; example of 'other': data-scope-content-type="newspaper_articles". Multiple facets may be combined, separated by commas. For example: data-scope-content-type="newspaper_articles,MEDIA"
data-scope-subj-terms
[comma separated values] - Subject terms to go with each search
data-scope-date
[YYYY:YYYY] - date in YYYY format ex: 2007:2015 searches between 2007 and 2015, 0:1996 searches from beginning to 1996, 1996:9999 will search after 1996.
data-scope-discipline
[discipline1,discipline2...] - comma separated disciplines, or a single discipline: data-scope-discipline="Computer Science"
data-scope-fulltext
true|false - include full text only. Not including this attribute defaults to FALSE
data-scope-scholarly
true|false - include scholarly only (Peer Review). Not including this attribute defaults to FALSE
data-scope-local
true|false - search within your catalog only. Not including this attribute defaults to FALSE
data-tagline
default|[custom text] - set as "default" or to your own custom text. If you don't include this attribute, or if you leave it empty ("") no text will display in its place.
data-placeholder
default|[custom text] - set as "default" or to your own custom text. If you don't include this attribute, or if you leave it empty ("") no text will display in its place.
data-advanced
default|[custom text] - set as "default" or to your own custom text. If you don't include this attribute, or if you leave it empty ("") no text will display in its place.
data-login
default|[custom text] - set as "default" or to your own custom text. If you don't include this attribute, or if you leave it empty ("") no text will display in its place. This will place a link to log into My Library Card under the search box.
data-custom-link
default|[custom link text] - set as "default" or to your own custom text. If you don't include this attribute the link will not appear. This will place either the link set in customization or specfied in the attribute below the search box behind any other links (such as Advanced or Login).
data-custom-link-url
[custom url] - set to your own custom text but will only be used if data-custom-link is set, even if default. If you don't include this attribute the default custom link url will be used if the data-custom-link attribute is set.

JavaScript Customization

The only variable you will need to change out of the box are the following:

discoveryCustomSettings.*

css
[path to css file] - where is the primo.css file located? Best practice is to start with // not http or https (//yoursite.edu/[pathtocssfile])
url
[https://your-primo.hosted.exlibrisgroup.com] - the base URL to your Primo site. This SHOULD NOT include primo-explore/search?vid=YOURVIEW as that will be added automatically by the search box.
instituion
INST_CODE - the institution code for Primo. You can find this in Primo Back Office.
vid
YOURVIEW - the view for the institution. You can usually find this in the URL after a search: &vid=YOURVIEW
tab
default_tab - the tab results should be in (usually default). You can usually find this in the URL after a search: &tab=default_tab
search_scope
yourcode - the search scope. You can usually find this in the URL after a search: &search_scope=yourcode.
bulkSize
10 - the number of results you want returned per page in Primo.
localParam
library|local|local2...etc - depends how you want local defined (campus, city, library branch, etc). If you have a facet for School, Library, or Location, use it and see what is listed in the URL: "&facet=library,include,LIBRARYCODE" (in this case it would be "library ", the first part of the comma delimited value, lower case). Another example: "&facet=local2,include,My Library" (in this case it would be "local2 ", the first part of the comma delimited value, lower case). Note, you can only define this once per script. So unless you are part of a consortium and want it scoped to only your school, or you are part of a larger library system but maintain a separate page for a particular branch, you probably won't use this.
localDesc
some code or string - this is paired with localParam and is the third value of the comma delimited set. For example, "&facet=library,include,LIBRARYCODE" (in this case it would be "LIBRARYCODE", the last part of the comma delimited value). Another example: "&facet=local2,include,My Library" (in this case it would be "My Library", the last part of the comma delimited value).
facet_books
[facet to map to] - what facet do you want "BOOKS" to map to? Default is "books" but this might not be a facet in your set-up.
facet_audio
[facet to map to] - what facet do you want "AUDIO" to map to? Default is "audio" but this might not be a facet in your set-up.
facet_video
[facet to map to] - what facet do you want "VIDEO" to map to? Default is "video" but this might not be a facet in your set-up.
facet_music
[facet to map to] - what facet do you want "MUSIC" to map to? Default is "score" but this might not be a facet in your set-up.
facet_media
[facet to map to] - what facet do you want "MEDIA" to map to? Default is "media" but this might not be a facet in your set-up.
custom_link_url
[url] - define a custom link to link to under the search box such as a feedback or ask us form. This will place it behind Advanced or Login if they are present, or alone if not. This can be overridden in the data-custom-link-url attribute, but only one custom link can be shown at a time.
custom_link_text
[text] - define the text to show for the custom link under the search box such as feedback or ask us. Can be overridden in the data-custom-link attribute.
narrow_max
380 is default - define what determines a narrow width for when the search box is in a narrow column or screen. The default display of the search box is for wide width (ideally > 400 pixels). The size of the search button with the text "Search" and the search icon in the left of the input field takes up too much space in narrow applications. So when a narrow width is encountered the icon on the left disappears for more text entry area, and the search button changes to a square button with just the search icon. Tip: If a search icon on as the button is always desired no matter the column width, set narrow_max to a really high number such as 9999 and it will always display this way.
